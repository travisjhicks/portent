# Examples

These examples show how Portent separates reality from views.

## Responsibility And Project

A responsibility describes a long-running outcome:

> Maintain personal health.

A project describes bounded output meant to improve that outcome:

> Start playing padel twice a week by the end of the quarter.

The project is temporary. The responsibility continues after the project ends.

## Event And People

A meeting is an event.

It may belong to a project, be related to several people, mention several topics, and produce follow-up tasks. It should not need to choose between a "meetings" folder and a "project" folder.

## Note And Topic

A note is a durable artifact.

It can document a project decision, explain an operation, or summarize research. Topics help collect notes without owning them.
